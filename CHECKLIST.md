# Completion Checklist

## Before Marking Done
- The code matches the user's request.
- The workspace structure was checked first.
- The change scope stayed minimal.
- Required files were updated only where needed.
- Validation was performed.
- Remaining risks were stated.

## UI Checklist
- Status is easier to read than decoration.
- Role cards show what each role is doing.
- Blockers are visible.
- Approval state is visible.
- Empty states are explicit.
- The layout works on mobile.

## Code Checklist
- No obvious runtime errors from missing data.
- No unchecked dynamic text injection.
- No hard-coded assumptions that can easily break.
- No unnecessary dependencies were added.
- Existing style and naming were respected.

## Verification Checklist
- File wiring checked.
- Behavior checked.
- Edge cases checked.
- Reviewer findings addressed or documented.
- Runtime limitations documented if test execution was not possible.
